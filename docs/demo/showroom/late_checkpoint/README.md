# Late Checkpoint

> This is a synthetic/sanitized public-safe showroom example. It is not generated by the public Action Preview and does not expose the private SDVM engine, scoring internals, calibration, thresholds, weights, statistical methodology, or playbook logic.

## What this scenario illustrates

Quality or coordination checks happen too late in the workflow. Corrections accumulate near the end, creating avoidable rework and making it harder to distinguish local errors from systemic process friction.

In this synthetic example, work skips expected intermediate checkpoints and moves toward finalize before review constraints are enforced. Problems discovered late force expensive backward movement through the workflow.

## PRE

Before adjustment, stages can advance without passing through agreed quality gates. Finalize-bound work sometimes bypasses review entirely, and corrections cluster at the end of the cycle.

See [`pre.md`](pre.md) for the PRE narrative.

## POST

After adjustment, mandatory checkpoints are enforced before finalize. Work must pass through review (or an equivalent agreed gate) with explicit transition evidence.

See [`post.md`](post.md) for the POST narrative.

## DELTA

The comparison shows fewer skip-ahead paths and less late-stage rework recovery. Some backward transitions may remain where legitimate exceptions exist.

See [`delta.md`](delta.md) for the DELTA narrative.

## Public-safe interpretation

Late checkpoint pathology is about **when** coordination checks occur, not whether checks exist. Public-safe reading focuses on whether evidence shows work reaching finalize without intermediate validation — a fragile transition pattern.

## How to discuss this with a pilot sponsor

1. **Name skip-ahead behavior** — finalize-bound work that bypasses agreed gates.
2. **Link late discovery to rework** — corrections cost more when found near the end.
3. **Agree on canonical path** — what must every run pass through?
4. **Plan PRE/POST capture** — after checkpoint enforcement, in controlled pilot conditions.
5. **Keep scope honest** — this gallery illustrates narrative shape, not engine output.

# Reviewer Loop

> This is a synthetic/sanitized public-safe showroom example. It is not generated by the public Action Preview and does not expose the private SDVM engine, scoring internals, calibration, thresholds, weights, statistical methodology, or playbook logic.

## What this scenario illustrates

A reviewer repeatedly asks for clarification or correction across comparable runs. The final output may still be acceptable, but the workflow absorbs increasing review effort because earlier stages fail to preserve enough context.

In this synthetic example, work oscillates between drafting and review. Each cycle adds coordination load even when the end artifact eventually ships.

## PRE

Before adjustment, draft and review stages ping-pong: submissions return for rework without durable context carry-forward. Review effort grows run over run.

See [`pre.md`](pre.md) for the PRE narrative.

## POST

After adjustment, earlier stages capture decisions and constraints more completely before review. Review cycles focus on substance rather than reconstructing missing context.

## DELTA

## Public-safe interpretation

A reviewer loop is **repeated rework** visible as a coordination pattern — not a judgment on reviewer thoroughness. Public-safe reading asks whether upstream work arrives at review with enough context to decide once, not whether review exists at all.

## How to discuss this with a pilot sponsor

1. **Describe the loop** — draft and review cycling without convergence.
2. **Separate quality from friction** — acceptable outputs can still hide expensive coordination.
3. **Ask about context capture** — what must draft include before review is fair?
4. **Propose PRE/POST windows** — after agreed upstream capture changes, in a controlled pilot.
5. **Anchor scope** — full SDVM analysis stays in authorized pilot work, not this gallery.

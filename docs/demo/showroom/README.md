## Purpose

This gallery provides **static, synthetic/sanitized** examples of how PRE, POST, and DELTA report **narratives** may be read when reviewing workflow friction patterns in a controlled comparison context.

These examples are presentation-oriented illustrations. They help readers understand common coordination friction patterns — handoff ambiguity, reviewer loops, and late checkpoints — without exposing the private SDVM engine.

## How to read these examples

Each scenario folder contains four Markdown files:

- `README.md` — overview and evidence follow-up framing
- `pre.md` — situation before a workflow adjustment
- `post.md` — situation after a workflow adjustment
- `delta.md` — qualitative comparison between PRE and POST

Read **PRE → POST → DELTA** in order. Each file uses qualitative language only: evidence patterns, public-safe readings, and explicit limitations.

> These examples are static, synthetic/sanitized, and presentation-oriented. They are not generated by the public Action Preview. They illustrate how PRE / POST / DELTA report narratives may be read in a controlled comparison context without exposing the private SDVM engine.

## Relationship to the SDVM walkthrough

Start with the [**SDVM Walkthrough**](../sdvm_walkthrough_demo.md) for the primary narrative: why workflow coordination friction matters, how evidence screening fits, and when a controlled pilot may be appropriate.

The showroom gallery is a **secondary, optional** layer. Use it after the walkthrough when a reader wants concrete examples of how PRE / POST / DELTA discussions might be structured.

## Relationship to the Action Preview

The [**Action Preview**](../../README.md) is the executable public-safe screening layer. It validates canonical JSONL evidence shape and produces limited preview reports (`validation.json`, `preview_report.md`).

The showroom gallery is **not** produced by the Action Preview. It does not demonstrate CLI or GitHub Action output. It illustrates how stakeholders might **read candidate readouts** and structure evidence follow-up questions — not how to run screening.

For a screening output illustration, see the [**friendly sample report**](../friendly_sample_report.md).

## Limitations

- No numeric diagnostic values, scoring tables, or causal claims appear in these examples.
- No scoring internals, statistical methodology, calibration, thresholds, weights, or internal recommendation logic are exposed.
- These are **not** full SDVM reports and **not** pilot-grade diagnosis.
- Synthetic scenarios only — no customer, sponsor, or production trace data.
- Visual HTML summaries are presentation derivatives only — not raw internal showroom exports or Action Preview output.
- No raw JSON report exports, metrics, audit variants, or baseline stores.

## Recommended use

1. Read the walkthrough first.
2. Optionally browse one showroom scenario relevant to your workflow concern.
3. Run the Action Preview on your own synthetic or authorized evidence when ready for executable screening.
4. Use the friendly sample report to illustrate screening output format.
5. Proceed to controlled pilot intake when evidence shape and stakeholder alignment support it: **https://sdvm.tech/pilot/intake/**
